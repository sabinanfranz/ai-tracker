# 20 — UX Design

## 개요

AI Update Tracker는 ChatGPT, Gemini, Codex, Claude Code 4개 제품의 공식 Release Notes를 세로 타임라인으로 통합하여 보여주는 공개 웹 트래커입니다. 사용자는 한눈에 최신 업데이트를 확인하고, 제품과 태그로 필터링하며, 상세 내용을 확인할 수 있습니다.

### Sticky Filter Bar (상단 고정)

```
┌─────────────────────────────────────────────────────────────────┐
│ [Product Filter]              [Tag Filter]       [Jump Controls] │
│                                                                   │
│ ☑ ChatGPT   ☐ Gemini        ☑ New Feature      << 2025 | 2026 >>│
│ ☑ Codex     ☐ Claude Code   ☐ Model             Latest >>       │
│                              ☐ Breaking                          │
│                              ☐ Deprecation       [🔄 Auto-refresh]│
│                              ☐ Access                            │
│                              ☐ Bugfix                            │
│                                                                   │
│ Filter Status: 2 Product × 1 Tag = 12 events shown / 89 total    │
└─────────────────────────────────────────────────────────────────┘
```

**설계 원리**:
- Sticky position으로 스크롤해도 항상 보임
- 각 필터 체크박스는 즉시 반영 (debounce 250ms)
- 선택된 필터 개수를 우측에 표시
- Year Jump는 앵커링으로 해당 월 구간으로 스크롤

### 이벤트 카드 (접힌 상태)

```
┌──────────────────────────────────────────────────────────────┐
│   ●  2026-03-01                                    [Severity] │
│   │                                                           │
│   │   [ChatGPT] [New Feature] [Model]                        │
│   │                                                           │
│   │   GPT-4o with Voice Enhanced Features                    │
│   │   Real-time conversation with advanced neural            │
│   │   processing and <500ms latency...                       │
│   │                                                           │
│   │   #voice #model #realtime                                │
│   │                                                           │
│   │   [View Details ▼]               [Source link →]         │
│   │                                                           │
└──────────────────────────────────────────────────────────────┘
```

**카드 내용 상세**:
- **Date**: 이벤트 날짜 (YYYY-MM-DD), 좌측에 선과 점으로 연결
- **Product Badge**: 제품명 배지 (4가지 색상 코딩)
- **Tags**: 최대 3개 테그 (파일링 라벨 형태, 오른쪽 정렬)
- **Title**: 한 줄, 굵은 글씨 (16px)
- **Summary**: 최대 2줄 (14px, 회색 #666)
- **Hashtags**: 관련 주제 태그 (3~5개, 파란색)
- **Actions**:
  - [View Details ▼] - 클릭하면 펼침
  - [Source link →] - 새 탭에서 원문 열기

**시각적 강조 (Severity 기반)**:
- **Severity 5 (Breaking)**: 좌측 보더 4px 빨강 (#EF4444), 배경 밝은 핑크 (#FEE2E2)
- **Severity 4 (New Model)**: 좌측 보더 4px 주황 (#F97316)
- **Severity 3 (Access)**: 좌측 보더 4px 파랑 (#3B82F6)
- **Severity 2 (Feature)**: 좌측 보더 4px 초록 (#22C55E)
- **Severity 1 (Bugfix)**: 좌측 보더 2px 회색 (#9CA3AF)

### Jump 컨트롤 (Sticky Bar 우측)

```
┌─────────────────────┐
│ << 2025 | 2026 >>   │
│ Latest >>           │
│ [🔄 Auto-refresh]   │
└─────────────────────┘
```

**상호작용**:
- `<< 2025`: 2025년 첫 이벤트로 스크롤 (smooth scroll)
- `2026`: 2026년 첫 이벤트로 스크롤
- `Latest >>`: 최신 이벤트로 스크롤 (top)
- `🔄`: 데이터 새로고침 (폴링 또는 수동)

## 인터랙션 명세

### 1. 필터 인터랙션 (Product & Tags)

**Product 필터**:
```
[✓ ChatGPT] [Gemini] [Codex] [Claude Code]

사용자 클릭:
1. 토글 상태 변경 (체크박스 토글)
2. 즉시 타임라인 업데이트 (debounce 250ms)
3. URL 쿼리 파라미터 업데이트: ?products=chatgpt,codex
4. 필터 상태 문자열 업데이트: "2 products selected"
5. 스크롤 위치: 현재 스크롤 위치 유지 (또는 top 이동)
```

**Tag 필터**:
```
[✓ New Feature] [✓ Model] [Breaking] [Deprecation] [Access] [Bugfix]

사용자 클릭:
1. 토글 상태 변경 (다중 선택)
2. 즉시 타임라인 업데이트 (debounce 250ms)
3. URL 쿼리 파라미터: ?tags=new_feature,model
4. 필터 상태: "6 tags available, 2 selected"
5. "Reset Tags" 링크 표시 (최소 1개 이상 선택 시)
```

**필터 조합 로직**:
- Product 필터: OR (ChatGPT OR Codex 선택 시, 둘 중 하나에 속한 이벤트 표시)
- Tag 필터: OR (New Feature OR Breaking 선택 시, 둘 중 하나 태그인 이벤트 표시)
- Product × Tag: AND (ChatGPT AND New Feature = ChatGPT에서 New Feature만 표시)

**Performance**:
- Debounce: 250ms (입력 중 불필요한 업데이트 방지)
- 필터링 결과: 클라이언트 사이드 (pre-loaded data 기반)
- 로드된 데이터 없으면: 서버에서 필터링 (lazy loading)

### 3. 스크롤 동작

**Lazy Loading (Infinite Scroll)**:
```
조건: 사용자가 페이지 bottom에서 300px 이내로 스크롤할 때

동작:
1. 로딩 인디케이터 표시: "[🔄 Loading more events...]"
2. 백그라운드에서 다음 배치(10개) 로드 (API 요청)
3. 로드 완료 후 스켈레톤 제거
4. 새 카드 append (DOM에 추가)
5. 사용자 스크롤 위치 보존

Debounce:
- 동시 다중 요청 방지 (request flag)
- API 호출 최소 간격: 500ms

Last Page:
- 모든 이벤트 로드됨 → "No more events" 메시지
```

**Jump 컨트롤 스크롤**:
```
사용자 클릭 [2025] 또는 [Latest]:
1. Smooth scroll 시작 (duration: 500ms)
2. 해당 month divider 또는 최신 이벤트로 이동
3. 이동 완료 후 해당 영역 하이라이트 (0.5초 동안 배경색 변경)

구현:
- JavaScript: window.scrollTo({ top: targetPosition, behavior: 'smooth' })
- 또는 Intersection Observer로 month divider 위치 추적
```

- 스크롤 후 해당 영역 하이라이트 (0.5초 배경색)
- Sticky bar에 "Current month: 2026-03" 표시

### 4. 카드 펼침 상태의 시각적 구분

**문제**: 펼쳐진 카드와 일반 카드의 구분이 약함

**해결**:
- 펼쳐진 상태: 좌측 보더 4px (제품 컬러)
- 펼쳐진 콘텐츠: 배경색 #F9FAFB (미묘한 회색)
- Evidence 영역: 배경색 #EFF6FF (매우 밝은 파랑)
- 닫기 버튼 우측 정렬 (명확한 CTA)

---

### 5. 모바일 터치 타겟

**문제**: 작은 화면에서 버튼 누르기 어려움

**해결**:
- 최소 터치 타겟: 44px × 44px (Apple HIG)
- 카드 전체: 클릭 가능 (펼치기)
- 체크박스: 더 큼 (24px × 24px)
- 간격: 최소 8px 여백

---

### 6. 네트워크 에러 복구

**문제**: 네트워크 끊김 시 사용자 경험 저하

**해결**:
- 3단계 재시도 로직:
  1. 즉시 재시도 ([Retry Now] 버튼)
  2. 자동 재시도 (30초 후)
  3. Offline mode: 캐시된 데이터 표시 + "Offline, showing cached data" 배너

**향후 추가**:

## 마치며

이 UX 문서는 AI Update Tracker의 MVP 버전을 위한 완전한 설계 사양입니다. 모든 상호작용은 사용자가 **10초 안에 최신 AI 업데이트를 파악할 수 있도록** 최적화되었습니다.

**주요 설계 원칙**:
1. **단순성**: 화면 1개, 핵심 기능만
2. **신속성**: 필터 → 스크롤 → 상세 → 소스 링크 (빠른 workflow)
3. **신뢰성**: 매 카드마다 소스 링크 & 근거 (Evidence)
4. **접근성**: WCAG AA 준수, 모바일 대응

프론트엔드 개발자는 이 문서를 기반으로 HTML/CSS/JavaScript를 구현하면 됩니다.
