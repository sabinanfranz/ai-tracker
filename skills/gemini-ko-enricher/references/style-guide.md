# Gemini 이벤트 한국어 Enrichment 스타일 가이드

## title_ko 규칙

### 어미
- **~해보세요!** / **~됩니다!** / **~만나보세요!** / **~받아보세요!** 등 권유·안내형 어미 사용
- 톤: 사용자에게 말을 거는 친근한 안내·권유형
- 절대 금지: ~습니다(딱딱), ~됨/~추가(명사 종결), ~하였음(보고서체)

### 길이
- 목표: 20~50자
- 최소 20자, 최대 50자

### 구조
- 핵심 혜택을 **사용자 관점의 문장**으로 전달
- 기능 나열이 아닌, 사용자 경험 중심 서술
- 한 문장으로 "이걸 쓰면 뭐가 좋은지"를 알 수 있어야 함

### 금지 패턴
- 키워드 나열: "A, B 및 C" / "A, B, C 지원"
- 명사 종결: "~출시" / "~추가" / "~제거"
- 개발 용어 직역: "비동기 압축 개선" → "대화가 더 빠르게 정리돼요"
- 물음표 어미: "~할까요?" / "~인가요?"

### 고유명사 규칙
- 서비스명: Gemini, Google, YouTube, Gmail, Google Maps, Google Workspace 등 영어 유지
- 기능명: Deep Research, Gems, Canvas, Google AI Studio 등 영어 유지
- 프로토콜/기술: API, MCP, PDF, CSV 등 영어 유지
- 모델명: Gemini 2.0 Flash, Gemini Ultra 등 영어 유지

## content_ko 규칙

### 형식
- 정확히 **3줄**, `\n`으로 구분
- 각 줄은 한 문장

### 각 줄의 역할
- **1줄 (변화)**: 이번에 어떤 변화가 생겼는지 쉽게 설명
- **2줄 (이점)**: 구체적으로 어떤 점이 좋아졌는지 안내
- **3줄 (활용)**: 사용자가 어떻게 활용하면 좋을지 권유

### 어투
- **~해요** / **~돼요** / **~있어요** / **~받아요** 체 (친근한 서술형)
- 절대 금지: ~습니다(딱딱), ~됨(명사종결)

### 길이
- 각 줄 20~80자

### 기술 용어 해설
- 첫 등장 시 괄호로 해설: "Deep Research(심층 분석 기능)", "Gems(맞춤형 AI 전문가)"
- 2회차부터는 이름만 사용

## 변환 사고 과정 (6단계)

1. **원문 읽기**: title + content 확인
2. **핵심 혜택 추출**: 사용자에게 어떤 가치를 주는지 파악
3. **사용자 관점 재구성**: 개발자 용어 → 일반 사용자가 이해할 수 있는 표현
4. **어미 결정**: 권유형(~해보세요!) 또는 안내형(~됩니다!)
5. **문장 완성**: 핵심 혜택 + 어미로 title_ko 작성
6. **길이 조정**: 20~50자로 다듬기

## content_ko 작성 사고 과정

1. title + content 정보 확인
2. 변경의 what(변화) / why(이점) / how(활용) 분리
3. 각 항목을 한 줄씩 작성
4. 기술 용어 첫 등장 시 괄호 해설 추가
5. 각 줄 20~80자 검증
6. 전체 톤 일관성 확인 (~해요/~돼요 체)
