# 한국어 로컬라이제이션 스타일 가이드

## title_updated_ko 규칙

### 어미
- **~해요** / **~됐어요** / **~있어요** / **~나왔어요** / **~시작돼요** 체 사용
- 절대 금지: ~습니다, ~하였음, ~됨 (딱딱한 어미)

### 길이
- 목표: 40자 내외 (최소 20자, 최대 50자)

### 구조
- 사용자 관점에서 **무엇이 좋아졌는지** 한 문장으로 전달
- 기능 나열이 아닌, 사용자 경험 중심 서술
- 쉼표로 2개 절을 연결하는 패턴 권장: "A하고, B할 수 있어요"

### 금지 패턴
- 키워드 나열: "A, B 및 C" / "A, B, C 지원"
- 명사 종결: "~출시" / "~개방" / "~추가"
- 개발 용어 직역: "비동기 압축" → "대화를 자동 압축하고"

### 고유명사
- 모델명: GPT-5.3-Codex, GPT-5.2-Codex 등 영어 유지
- 프로토콜/도구명: MCP, API, SDK, CLI, IDE 등 영어 유지
- 서비스명: Slack, GitHub, Linear 등 영어 유지

## content_updated_ko 규칙

### 어미
- **~습니다** 체 (존댓말 서술체)

### 구조
- 정확히 3문장
- 1문장: 핵심 변경 사항
- 2문장: 부가 기능 또는 세부 개선
- 3문장: 적용 범위 또는 버그 수정

### 길이
- 각 문장 40~80자

## 변환 사고 과정

1. 원문(title_updated) 읽기
2. 핵심 변경 1~2개 추출
3. 사용자에게 어떤 영향인지 재구성
4. ~해요/~됐어요 어미로 문장 완성
5. 40자 내외로 다듬기
